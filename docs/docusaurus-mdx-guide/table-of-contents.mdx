# Table of Contents

Two ways to list table of contents in the same document:
1. List the whole contents within the same document.
2. List the partial contents within the same document. 

<br />

Below is auto listed TOC for this document
<TOCInline toc={toc} />

It is also possible to display an inline table of contents directly inside a Markdown document. This is the simplest way and is autogenerated on usage of `TOCInline` component from docusaurus. 

```markdown
## Dummy Heading 2 {#section1}
### Dummy Heading 3 {#section1.1}
#### Dummy Heading 3.1 {#section1.1.1}
### Dummy Heading 4 {#section2.1} 
#### Dummy Heading 4.1 {#secion2.1.1}
```

## Dummy Heading 2 {#section1}
### Dummy Heading 3 {#section1.1}
#### Dummy Heading 3.1 {#section1.1.1}
### Dummy Heading 4 {#section2.1} 
#### Dummy Heading 4.1 {#secion2.1.1}

```jsx
import TOCInline from '@theme/TOCInline';

<TOCInline toc={toc} />
```

:::info How to control the depth of TOC?

* The `:depth:` param or can be controlled by `toc_min_heading_level` and `toc_max_heading_level` meta for individual files and `minHeadingLevel` and `maxHeadingLevel` for global theme config in docusaurus as given in docs.
* The `:backlinks:` seems to be set to none for all and also there is no direct replacement in MDX via Docusaurus. So, can be ignored.
* The `:local:` can be ignored as well cause the TOCInline component works only for local (same) document.

:::

The `toc` can easily be cut out to exclude unwanted nodes or insert extra nodes, and create a new TOC tree.

```markdown
<TOCInline
  // Only show h2 and h4 headings
  toc={toc.filter((node) => node.level === 2 || node.level === 4)}
  minHeadingLevel={2}
  // Important: Show h4 headings in addition to the default h2 and h3 headings
  maxHeadingLevel={4}
/>
```
This is how it be rendered: 

<TOCInline
  // Only show h2 and h4 headings
  toc={toc.filter((node) => node.level === 2 || node.level === 4)}
  //minHeadingLevel={2}
  // Important: Show h4 headings in addition to the default h2 and h3 headings
  maxHeadingLevel={4}
/>

Please refer docusaurus [Inline TOC docs](https://docusaurus.io/docs/markdown-features/inline-toc) for more insights into syntax and usage of `TOCInline`.

## Handling custom table of contents

If one individual section requires to have its own TOC. For example, for creating TOC for subheading for `Dummy Heading 2` section.

Use the level to create the TOC as shown below:
```markdown
<TOCInline
  toc={
    // Only show section1.1 and section2.1 and thier next level headings
    [toc[1], toc[2], toc[3], toc[4]]
  }
  maxHeadingLevel={4}
/>
```

<TOCInline
  toc={
    // Only show 4th and 5th level heading
    [toc[1], toc[2], toc[3], toc[4]]
  }
  maxHeadingLevel={4}
/>

This can be achived by filtering `toc` with the heading id. Use the following to filter based on heading id:
```markdown
<TOCInline
  // Only show section1.1 and section2.1 headings
  toc={toc.filter((node) => node.id === 'section1.1' || node.id === 'section2.1')}
/>
```
Rendered as:
<TOCInline
  // Only show h2 and h4 headings
  toc={toc.filter((node) => node.id === 'section2.1' || node.id === 'section1.1')}
/>
